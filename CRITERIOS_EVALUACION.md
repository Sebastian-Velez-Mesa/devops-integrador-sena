# Criterios de Evaluación - Actividad Integradora DevOps

**Evaluador:** Instructor SENA  
**Programa:** DevOps y Contenedores (Docker)  
**Duración:** 30 minutos por equipo  
**Modalidad:** Trabajo colaborativo

---

## Fase 1: Introducción del Proyecto (5 minutos)

### Aspectos a evaluar:

| Aspecto | Descripción | Puntaje |
|---------|-------------|---------|
| **Nombre del Proyecto** | Claridad y relación con DevOps | 1 |
| **Problema Resuelto** | Explicación clara de la problemática abordada | 1 |
| **Objetivo General** | Alineación con cultura DevOps | 1 |
| **Usuarios Beneficiados** | Identificación correcta de stakeholders | 1 |
| **Tecnologías Utilizadas** | Listado y justificación inicial | 1 |
| **Presentación Verbal** | Claridad, seguridad y lenguaje técnico apropiado | 1 |

**Subtotal Fase 1: 6 puntos**

---

## Fase 2: Arquitectura del Proyecto (10 minutos)

### Aspectos a evaluar:

| Aspecto | Descripción | Puntaje |
|---------|-------------|---------|
| **Diagrama Arquitectura** | Claridad y completitud del diagrama | 2 |
| **Componentes Identificados** | Frontend, Backend, Docker, Docker Compose, GitHub | 2 |
| **Flujo de Información** | Explicación clara del flujo usuario → frontend → backend | 2 |
| **Decisiones Técnicas** | Justificación de tecnologías (Flask, Nginx, etc.) | 2 |
| **Mejores Prácticas DevOps** | Aplicación de principios (automatización, CI/CD, contenedores) | 1 |
| **Documentación Código** | Claridad del Dockerfile, docker-compose.yml, workflows | 1 |

**Subtotal Fase 2: 10 puntos**

---

## Fase 3: Demostración en Vivo (10 minutos)

### Aspectos a evaluar:

| Aspecto | Descripción | Puntaje |
|---------|-------------|---------|
| **Aplicación Funcionando** | Sitio web/API responde correctamente | 2 |
| **Contenedores Activos** | `docker ps` muestra contenedores corriendo | 1.5 |
| **Imágenes Docker** | `docker images` muestra imágenes construidas | 1.5 |
| **Redes Docker** | `docker network ls` y explicación de conectividad | 1 |
| **Volúmenes Docker** | `docker volume ls` y demostración de persistencia | 1 |
| **Repositorio GitHub** | Acceso, estructura y documentación | 1 |
| **Historial de Commits** | `git log` muestra evolución del proyecto | 1 |
| **Branches** | `git branch -a` y explicación de estrategia de ramas | 1 |
| **Docker Compose** | Demostración de `docker-compose up`, logs, dependencies | 1 |
| **Pipeline CI/CD** | Mostrar GitHub Actions, ejecución y artefactos | 1 |
| **Health Checks** | Demostración de endpoints `/health` funcionando | 1 |

**Subtotal Fase 3: 12 puntos**

---

## Fase 4: Conclusiones (5 minutos)

### Aspectos a evaluar:

| Aspecto | Descripción | Puntaje |
|---------|-------------|---------|
| **Dificultades Encontradas** | Honestidad y claridad en obstáculos | 1.5 |
| **Soluciones Implementadas** | Explicación de cómo se resolvieron | 1.5 |
| **Aprendizajes Adquiridos** | Reflexión sobre conocimientos consolidados | 1 |
| **Mejoras Futuras** | Propuestas viables y alineadas con DevOps | 1 |
| **Resumen Ejecutivo** | Síntesis del proyecto y su impacto | 1 |

**Subtotal Fase 4: 6 puntos**

---

## Entregas Requeridas

| Entrega | Descripción | Estado |
|---------|-------------|--------|
| Proyecto funcional | Código completo, repositorio actualizado | ✓ |
| Repositorio GitHub | Accesible, bien documentado | ✓ |
| Dockerfiles | Backend y Frontend optimizados | ✓ |
| Docker Compose | Multicontenedor con networks y volumes | ✓ |
| GitHub Actions | Pipeline CI/CD configurado | ✓ |
| Presentación PPTX/PDF | 30 minutos de cobertura | Pendiente |
| Evidencias de Ejecución | Capturas de pantalla, logs, demo script | ✓ |
| Documento Técnico | Decisiones de diseño y justificación | Pendiente (usuario) |

## Calificación Total

**Máximo puntaje: 34 puntos**

- **Fase 1 (Introducción):** 6 puntos (17.6%)
- **Fase 2 (Arquitectura):** 10 puntos (29.4%)
- **Fase 3 (Demostración):** 12 puntos (35.3%)
- **Fase 4 (Conclusiones):** 6 puntos (17.6%)

| Rango | Calificación |
|-------|-------------|

| 26-29 | Bueno (4.0) |

## Competencias Evaluadas

✓ **Cultura DevOps:** Aplicación de principios de automatización y colaboración.  
✓ **Contenedores Docker:** Construcción y uso de imágenes y contenedores.  
✓ **Redes y Volúmenes:** Administración correcta de conectividad y persistencia.  
✓ **Orquestación:** Docker Compose para servicios multicontenedor.  
✓ **Control de Versiones:** Git y GitHub para gestión de código.  
✓ **CI/CD:** Automatización de procesos con GitHub Actions.  
✓ **Argumentación Técnica:** Justificación de decisiones de diseño.  
✓ **Trabajo Colaborativo:** Desempeño en equipo bajo enfoque DevOps.

**Nota:** Esta rúbrica es orientativa. El instructor puede ajustar puntajes según el desempeño específico del equipo.
