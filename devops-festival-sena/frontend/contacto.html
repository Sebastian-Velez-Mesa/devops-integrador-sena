Preguntas Frecuentes

Utiliza una red aislada de tipo bridge llamada devops-network. El frontend (servido por Nginx) se conecta con el backend (API Flask) usando el nombre de servicio del contenedor dentro de la red privada, evitando exponer puertos innecesarios a la red pública.
Los datos enviados mediante este formulario o la reserva de tickets se escriben en contactos.json. Este archivo reside en el volumen con nombre devops-backend-logs, persistiendo la información aun si el contenedor se destruye o se actualiza.
El flujo de CI/CD realiza el chequeo de sintaxis del código Python, ejecuta pruebas automatizadas y realiza un build simulado de las imágenes del Frontend y Backend para asegurar que la aplicación compile sin problemas antes de la entrega final.
Este es un proyecto académico de sustentación técnica de DevOps desarrollado por los aprendices del SENA Centro de Tecnología y Manufactura Avanzada (CTMA), en el programa de formación de DevOps y Contenedores Docker.
